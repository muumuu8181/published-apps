## 発生した問題・解決記録（最重要）

### 問題1: Web Audio APIのコンテキスト初期化
- **発生タイミング**: Phase 2実装初期
- **問題内容**: ブラウザのAutoplay policyによりユーザーアクションなしでAudioContextが開始できない
- **解決方法**: 
  ```javascript
  // ユーザーアクションでAudioContextを明示的に再開
  if (audioContext.state === 'suspended') {
      await audioContext.resume();
  }
  ```
- **再発防止策**: Web Audio APIを使用する際は必ずユーザーアクション後にコンテキストを初期化

### 問題2: ADSRエンベロープの正確な制御
- **発生タイミング**: Phase 2実装中
- **問題内容**: exponentialRampToValueAtTimeが0値で失敗する問題
- **解決方法**:
  ```javascript
  // 0値を避けるため微小値（0.0001）を使用
  gainNode.gain.exponentialRampToValueAtTime(Math.max(0.0001, sustainLevel), currentTime + attack + decay);
  ```
- **再発防止策**: 指数関数的変化を使用する際は必ず正の値を保証する

### 問題3: リアルタイム波形描画のパフォーマンス
- **発生タイミング**: Phase 2実装中
- **問題内容**: 高頻度でのCanvas描画によるCPU負荷とちらつき
- **解決方法**:
  - requestAnimationFrame使用による60fps制限
  - AnalyserNodeのFFTサイズ最適化（2048）
  - Canvas描画の効率化（clearRect + stroke最適化）
- **再発防止策**: リアルタイム描画は必ずrequestAnimationFrameで制御し、描画頻度を適切に管理

## 発生した問題・解決記録（最重要）

### 問題1: A*アルゴリズムのヒューリスティック関数設計
- **発生タイミング**: Phase 2実装中
- **問題内容**: グラフの座標系とアルゴリズムの距離概念の不整合
- **解決方法**: 
  ```javascript
  // ユークリッド距離を正規化してヒューリスティックとして使用
  heuristic(a, b) {
      const nodeA = this.visualizer.nodes[a];
      const nodeB = this.visualizer.nodes[b];
      return Math.sqrt((nodeA.x - nodeB.x) ** 2 + (nodeA.y - nodeB.y) ** 2) / 50;
  }
  ```
- **再発防止策**: 座標系と論理的距離の適切な変換係数を事前に検討

### 問題2: 優先度キューの効率的実装
- **発生タイミング**: Phase 2実装中
- **問題内容**: JavaScript標準にない優先度キューの実装が必要
- **解決方法**: 
  ```javascript
  // 配列ベースの優先度キューを実装
  enqueue(element, priority) {
      const queueElement = { element, priority };
      // 挿入ソートで適切な位置に配置
      for (let i = 0; i < this.items.length; i++) {
          if (queueElement.priority < this.items[i].priority) {
              this.items.splice(i, 0, queueElement);
              return;
          }
      }
      this.items.push(queueElement);
  }
  ```
- **再発防止策**: 大規模グラフでは二分ヒープベース実装を検討

### 問題3: Canvas描画のパフォーマンス最適化
- **発生タイミング**: Phase 2実装中
- **問題内容**: エッジ重複描画とノード更新時のちらつき
- **解決方法**:
  - 描画前の完全クリア＋一括再描画
  - エッジ→ノードの順序で描画して重なり制御
  - ノード状態変更時の滑らかな色遷移
- **再発防止策**: Canvas描画は状態変更時にのみ実行し、不要な再描画を避ける

# 要件定義 - グラフアルゴリズムビジュアライザー

## アプリ番号
[00000076]

## アプリ名
グラフアルゴリズムビジュアライザー

## 概要
グラフ理論の主要アルゴリズム（BFS、DFS、Dijkstra、A*）を
ステップバイステップで実行・可視化するWebアプリケーション。
ランダムグラフ生成とカスタムグラフ入力に対応し、
アルゴリズムの動作原理を直感的に学習できる教育ツール。

## 機能要件

### 1. グラフ構造管理システム
- **データ構造**
  - 隣接リスト形式での内部表現
  - ノード最大数: 50（パフォーマンス考慮）
  - 重み付きエッジ対応（1〜20の整数値）
- **グラフ生成機能**
  - Erdős-Rényiモデルによるランダム生成
  - ノード数調整: 5〜50（スライダー制御）
  - エッジ密度調整: 0.1〜0.8（接続確率）
  - 重み範囲設定: 1〜20
- **手動入力機能**
  - 隣接リスト形式テキスト解析
  - フォーマット例: "0: 1(5), 2(3)"
  - 構文エラーハンドリング

### 2. アルゴリズム実装
- **幅優先探索（BFS）**
  - キューベースの実装
  - 最短パス長の計算
  - 未重み付きグラフ用最適経路探索
- **深さ優先探索（DFS）**
  - スタックベースの実装
  - 探索順序の可視化
  - 経路発見（最短経路保証なし）
- **ダイクストラ法**
  - 優先度キューによる実装
  - 重み付きグラフ最短経路
  - 距離更新の動的表示
- **A*アルゴリズム**
  - ヒューリスティック探索
  - f(n) = g(n) + h(n)の可視化
  - ユークリッド距離ヒューリスティック

### 3. ステップバイステップ実行制御
- **実行モード**
  - 自動実行（速度調整可能: 100ms〜2000ms）
  - 手動ステップ実行
  - 一時停止・再開機能
  - リセット機能
- **状態管理**
  - 各ノードの状態追跡（未訪問・訪問済み・処理中・キュー内）
  - エッジ状態管理（通常・最短パス）
  - アルゴリズム進捗の可視化

### 4. 可視化システム
- **Canvas描画エンジン**
  - 高品質2D描画（アンチエイリアス）
  - 60fps滑らかアニメーション
  - レスポンシブ対応
- **ノード可視化**
  - 状態別色分け（6種類の状態）
  - ノードID表示
  - 距離値表示（Dijkstra・A*用）
  - 影効果による立体感
- **エッジ可視化**
  - 重み値の中央表示
  - 最短パスのハイライト表示
  - 方向性の明確な描画
- **アニメーション効果**
  - 状態遷移のスムーズな色変化
  - パス発見時のハイライト効果
  - 進捗バーアニメーション

### 5. インタラクティブUI
- **ノード選択**
  - マウスクリックによる開始ノード設定
  - Shift+クリックによる終了ノード設定
  - 視覚的フィードバック
- **パラメータ制御**
  - リアルタイムスライダー調整
  - アルゴリズム選択ドロップダウン
  - 実行速度制御
- **制御ボタン**
  - 開始・一時停止・ステップ・リセット
  - グラフ生成・クリア機能
  - 手動入力・読み込み機能

### 6. 統計・分析機能
- **実行統計**
  - ステップ数カウント
  - 訪問済みノード数
  - キュー/スタックサイズ
  - 最短距離表示
  - 実行時間計測
- **進捗可視化**
  - プログレスバー表示
  - 完了率パーセンテージ
  - ステップ情報詳細表示
- **比較分析**
  - アルゴリズム間の性能比較
  - 統計データの保持・表示

### 7. 教育支援機能
- **ステップ解説**
  - 各ステップの動作説明
  - アルゴリズム固有の情報表示
  - 現在の処理内容説明
- **視覚的学習支援**
  - 色分けによる状態理解
  - 凡例表示
  - アルゴリズム特性の視覚的表現
- **カスタマイズ性**
  - 学習目的に応じたグラフ作成
  - 特定ケースの動作確認
  - 実験的学習環境

## 非機能要件

### 1. パフォーマンス要件
- **応答性**: UI操作0.1秒以内の反応
- **描画性能**: 60fps滑らかアニメーション
- **メモリ効率**: 最大50ノードで安定動作
- **実行効率**: アルゴリズム1ステップ50ms以内

### 2. スケーラビリティ
- **グラフサイズ**: 最大50ノード・1000エッジ
- **アルゴリズム拡張**: 新アルゴリズム追加容易性
- **UI拡張**: 新機能追加の柔軟性

### 3. ブラウザ互換性
- **対応ブラウザ**: Chrome, Firefox, Safari, Edge最新版
- **Canvas API**: 2D描画機能フル活用
- **JavaScript**: ES6+機能使用
- **レスポンシブ**: デスクトップ・タブレット対応

### 4. 技術仕様
- **フロントエンド**: 純粋JavaScript（ES6+）
- **描画**: Canvas API
- **データ構造**: Map、Set、Array、優先度キュー
- **UI**: CSS3（Grid Layout、Flexbox）

## 制約事項
- **グラフサイズ制限**: パフォーマンス維持のため最大50ノード
- **ブラウザ依存**: Canvas APIサポート必須
- **メモリ制限**: 大規模グラフでのメモリ使用量考慮
- **計算複雑性**: O(n²)を超える処理の最適化必要

## 使用シナリオ
1. **アルゴリズム学習**: 大学・専門学校での教育利用
2. **研究用途**: アルゴリズム動作の詳細分析
3. **実験環境**: 特定グラフでの動作確認
4. **デモンストレーション**: アルゴリズム説明用ツール

## 成功基準
- 全4アルゴリズムが正確に動作
- ステップバイステップ実行が直感的
- ランダムグラフ生成が多様性を持つ
- 手動入力が柔軟に対応
- 統計情報が教育的価値を提供
- 美しく理解しやすい可視化を実現
- レスポンシブデザインで多デバイス対応

## 拡張可能性
- 新アルゴリズム追加（ベルマン・フォード、フロイド・ワーシャル）
- 3D可視化対応
- グラフの自動レイアウト機能
- アルゴリズム性能ベンチマーク機能
- エクスポート・インポート機能
